# 生成项目文档

基于前两步的项目摘要和问题清单，生成以下三个文件：

---

## 1. README.md

结构如下（如果某项信息不确定，用 `<!-- TODO: 补充 -->` 占位，不要编造）：

- 项目名 + 一句话描述
- 截图/演示链接（如有）
- 功能列表（已完成 ✅ / 开发中 🚧）
- 技术栈
- 快速开始（Prerequisites → 安装 → 配置 → 启动）
- 项目结构说明（只列重要目录）
- 已知问题 / Roadmap（来自第二步的清单）

语气：面向 **GitHub 上的技术开发者**，简洁、直接。

## 2. SETUP.md

只写启动和部署相关的内容，面向「第一次接手这个项目的人」：
- 环境要求
- 本地开发启动步骤（每步一条命令）
- 环境变量说明（每个变量的用途）
- 常见报错和解决方法（如果能推断的话）

## 3. TODO.md

来自第二步的问题清单，格式：
- 🔴 阻塞问题
- 🟡 待完善项
- 💡 未来可做（如果第一步对话中提到了未完成功能）
