# 项目健康检查

基于已有的项目摘要，对代码库做以下检查，**每项都要给出具体文件和行号**：

## 检查清单

**依赖与配置**
- [ ] 有没有 package.json / requirements.txt 但实际缺失的依赖？
- [ ] .env.example 是否存在？有没有硬编码的密钥或路径？
- [ ] Node/Python 版本是否有限定？

**代码质量**
- [ ] 搜索所有 TODO / FIXME / HACK / XXX 注释
- [ ] 有没有明显的 console.log 调试代码残留？
- [ ] 有没有被注释掉的大段代码？

**可运行性**
- [ ] 入口文件能找到吗？启动命令是否可以推断出来？
- [ ] 有没有明显的未处理异常或裸 throw？
- [ ] 测试文件存在吗？能运行吗？

## 输出格式

输出两份清单：
1. **🔴 阻塞问题**（别人clone下来跑不起来的）
2. **🟡 待完善项**（能跑但不完整的）

每项格式：`- [ ] 问题描述 → 文件路径:行号`
